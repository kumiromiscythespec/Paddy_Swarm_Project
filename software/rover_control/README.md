# Paddy Swarm Rover Control Software

この文書は、Paddy Swarm のローバー操作ソフトウェアを今後開発するための境界と原則を定義する入口です。現在は設計・試作段階であり、実装コード、実機モーター出力、通信方式、データ形式はまだ確定していません。この文書は、実田んぼでの安全な稼働や無人運転を保証するものではありません。

## 1. 目的

- スマートフォンからローバーを安全に手動操作する。
- 最初は1台の仮想ローバーを操作する。
- 仮想試験後に ESP32 と MD10C へ接続する。
- 命令とテレメトリーを記録する。
- 通信不能時でもローバー側が単独で安全停止する。
- 完全自動農業システムを最初から作るものではない。

## 2. 現在の対象範囲

初期対象は次のとおりです。

- スマートフォン用ローカル PWA またはローカル Web UI
- 共通通信プロトコル
- 仮想ローバー
- ESP32 ローバーファームウェア
- 安全状態機械
- ログ形式
- テスト
- ハードウェア配線資料

最初の片側モーター試験で扱う命令は、最低限次のものとします。

- `ARM`
- `DISARM`
- `MOVE_FORWARD`
- `MOVE_REVERSE`
- `STOP`
- `EMERGENCY_STOP`
- `SET_SPEED_LIMIT`

## 3. 現段階の対象外

- クラウド管理サイト
- Google ログイン
- Google Earth 連携
- 農地座標のクラウド保存
- 完全自動運転
- 複数田んぼのクラウド統合管理
- 無人での実田んぼ作業
- 仮想ローバー試験前の実モーター接続

## 4. 安全原則

この節とリポジトリ共通の [`docs/safety.md`](../../docs/safety.md) をあわせて参照してください。両者に差異がある場合は、より安全側の制約を採用します。

- 押している間だけ動くデッドマン方式とする。
- 操作者が指を離したら停止する。
- アプリがバックグラウンドへ移動したら停止要求を行う。
- 通信が約 0.5〜1 秒途切れた場合、ESP32 側で停止する。正確な閾値は検証後に決める。
- 通信再接続後に自動で動作を再開しない。
- 起動直後は必ず `DISARM` とする。
- `ARM` は人間が明示的に行う。
- 古い命令を再実行しない。
- 各命令は連番、時刻、有効期限を持つ設計とする。
- `STOP` と `EMERGENCY_STOP` を区別する。
- 非常停止の解除だけでは動作を再開しない。
- PTO 動作中は走行命令を拒否する。
- 走行中は PTO の起動を拒否する。
- ハードウェア非常停止は、アプリや ESP32 の通常制御から独立させる。
- 実機出力は初期状態で無効にする。
- 安全判断の最終責任はスマートフォン UI ではなく ESP32 側に持たせる。
- 仮想ローバー試験を通過するまで実モーターを接続しない。

ここに記載した原則は設計要求であり、未検証の安全機能を保証するものではありません。

## 5. 開発順序

1. スマートフォン手動操作 UI
2. 仮想ローバー
3. 通信断停止
4. 実機モーター制御
5. テレメトリー記録
6. 片側クローラー試験
7. 左右クローラー制御
8. 簡易統制ステーション
9. 部分自動化

実機試験は段階的に行い、最初から実田んぼへ投入しません。

## 6. 推奨アーキテクチャ

仮想試験と実機試験は、できる限り同じ通信契約を使用します。

```text
スマートフォン PWA
    → 共通プロトコル
仮想ローバー
```

```text
スマートフォン PWA
    → 同じ共通プロトコル
ESP32 実機
```

UI、プロトコル、安全状態機械、シミュレーター、ファームウェア、ログ形式は分離します。これにより、未確定の通信方式や実装技術を後から変更できる境界を保ちます。

## 8. 各領域の責務

- `apps/mobile_pwa/`: スマートフォン操作 UI
- `firmware/esp32_rover/`: 実機出力と安全停止
- `protocol/`: 共通通信契約
- `simulator/virtual_rover/`: 実機と同じ通信契約を使う仮想ローバー
- `safety/`: 状態機械と安全要件
- `logs/`: バージョン管理されたログ形式
- `shared/`: 同一言語間で共有可能な共通処理
- `config/`: 機種依存値と安全な設定例
- `tests/`: プロトコル、安全、統合試験
- `docs/hardware/`: ESP32、MD10C、非常停止、リレー、ヒューズ、電源、エンコーダーの配線資料

## 9. バージョン管理方針

```text
Common Rover hardware: v2.28.2
Mobile PWA:            独立した SemVer
ESP32 firmware:        独立した SemVer
Virtual Rover:         独立した SemVer
Protocol:              v0 から開始
Log format:            v0 から開始
```

Common Rover のハードウェア版番号を、アプリやプロトコルの版番号として流用しません。また、`mobile_pwa_v0_1` のようにソースフォルダ名へ版番号を埋め込まず、Git タグや将来の manifest で管理します。

## 10. 設定と秘密情報

次の情報は Git に保存しません。

- Wi-Fi SSID
- Wi-Fi パスワード
- API キー
- 秘密鍵
- 個人情報
- 正確な農地座標
- 実機固有の秘密設定

機種依存値はコードへ直接ハードコードせず、設定として分離します。安全なサンプル値のみを `config/examples/` に置く方針とし、秘密情報や実在地点を含めません。
