# CONTRIBUTING.md

# Paddy Swarm Project への参加方法

Paddy Swarm Project / 田んぼロボット群プロジェクトに関心を持っていただき、ありがとうございます。

このプロジェクトは、小規模・家族経営の稲作現場で、人間の負担が大きい作業を小型ロボット群で少しずつ減らすことを目指す、実験・試作段階のオープンなプロジェクトです。

現在の公開段階は **Grade 0** です。

Grade 0では、完成ロボットを提供するのではなく、3Dプリント用データ、CAD、試験手順、失敗ログ、組み立て説明書を公開し、協力者が小さく安全に試せる状態を目指します。

---

## 最重要方針

```text
実田んぼは収入源であり、失敗してよい実験場ではありません。
```

いきなり実田んぼに投入せず、必ず以下の順で試験してください。

```text
机上
↓
乾いた地面
↓
水槽
↓
柔らかい土
↓
浅泥
↓
水あり模擬環境
↓
小区画・非本番環境
↓
実田んぼ端部
```

## Issueの投稿方法

問題報告、印刷結果、改善案、アイディアはGitHub Issuesへ投稿してください。

### Issueタイトル例

```text
[print] スナップ爪が折れた
[fit] スライドレールがきつすぎる
[float] フロートが片側に傾く
[water] 傾斜時に防水BOX位置へ水が来る
[docs] 組み立て説明が分かりにくい
[idea] 直播アーム案
[idea] 高刈りアーム案
[safety] 危険な構造を見つけた
[field] 水口構造の現場メモ
```

### Issueに書いてほしい情報

## Pull Requestの方針

Pull Requestを歓迎します。

歓迎する変更：

- CADデータの改良
- STLの改良
- 説明書の改善
- 試験ログの追加
- 失敗ログの追加
- 安全注意の追加
- 部品表の整理
- ファイル名の整理
- 図解の追加
- GitHub構成の改善
- ソフトウェアの小さな改善

Pull Request前に、以下を確認してください。

```text
1. 変更内容が分かる説明を書いた
2. 安全性に関わる変更か確認した
3. 既存ファイル名・部品番号ルールに沿っている
4. 印刷条件や試験条件を書いた
5. 写真に個人情報・正確な位置情報が写っていない
6. 実田んぼでの危険な試験を推奨していない
7. 農薬・電気・機械安全に関して断定しすぎていない
```

## ファイル名ルール

部品ファイルは、できるだけ以下の形式にしてください。

```text
PSR_<カテゴリ>_<番号>_<部品名>_v<バージョン>.<拡張子>
```

例：

```text
PSR_CONN_001_snap_fit_test_v0_1.stl
PSR_CONN_002_slide_rail_test_v0_1.stl
PSR_FLOAT_001_side_float_section_v0_1.stl
PSR_GB_001_gearbox_cut_model_v0_1.stl
PSR_HC_001_high_cut_arm_base_v0_1.stl
```

---

## 写真・動画の共有ルール

写真や動画は非常に役立ちます。

ただし、以下に注意してください。

- 正確な住所や位置情報を含めない
- 家族や近隣住民が写り込まないようにする
- ナンバープレートを写さない
- 田んぼの場所が特定されすぎないようにする
- 危険な試験を成功例のように見せない
- 失敗写真も歓迎する

## 安全に関わる変更

以下に関係する変更は、安全上の確認を優先します。

- 防水BOX
- バッテリー
- 充電
- モーター
- 刃物
- 回転部品
- 高刈り収穫アーム
- 除草撹拌アーム
- 直播アーム
- 水管理ゲート
- 実田んぼ投入手順
- 農薬・種子処理
- 家畜飼料利用

安全に関わる提案は、すぐに実装するより、まずIssueで議論してください。

## コミュニケーション方針

Paddy Swarm Projectでは、現場経験、試作経験、失敗経験を尊重します。

歓迎するもの：

- 小さな実験
- 失敗報告
- 現場からの違和感
- 安全上の指摘
- コスト削減案
- 修理しやすさの提案
- 農家目線の使いにくさ指摘

避けたいもの：

- 危険な試験の推奨
- 実田んぼへの無責任な投入提案
- 他者の失敗を責める発言
- 高価すぎる前提の提案だけを押し付けること
- 農業現場の制約を無視した設計
- 安全確認なしの刃物・電装・水中試験

## 最後に

Paddy Swarm Projectは、完成品を急ぐプロジェクトではありません。

まずは、机の上で壊す。  
水槽で沈める。  
模型で問題を見つける。  
そのうえで、田んぼへ少しずつ近づける。

安全に、小さく、現実的に試していきましょう。
